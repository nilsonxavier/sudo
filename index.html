Bem-vindo à Páginas GitHub.

Este gerador automático de página é a maneira mais fácil de criar belas páginas para todos os seus projetos. Autor o conteúdo da página aqui, usando GitHub Flavored Markdown, selecione um modelo criado por um designer e publicar. Depois de sua página é gerada, você pode conferir o novo ramo:

$ cd your_repo_root/repo_name
$ git fetch origin
$ git checkout gh-pages

Se você estiver usando o GitHub para Mac, simplesmente sincronizar seu repositório e você verá o novo ramo.

Modelos de grife

Nós temos trabalhado alguns modelos bonitos para você usar. Vá em frente e continuar a layouts para navegar por elas. Você pode facilmente voltar a editar a sua página antes de publicar. Após a publicação de sua página, você pode revisitar o gerador de página e mudar para outro tema. O conteúdo da página será preservada se permanecesse formato remarcação.

Em vez Dirija Stick?

Se você preferir não usar o gerador automático, aperte um ramo chamado gh-páginas para o seu repositório para criar uma página manualmente. Além de apoiar o conteúdo HTML regular, Páginas GitHub apoiar Jekyll, um blog ciente gerador simples, site estático escrito por nosso próprio Tom Preston-Werner. Jekyll torna mais fácil para criar cabeçalhos e rodapés de todo o site, sem ter que copiá-los através de cada página. Ele também oferece suporte blogue inteligente e outros recursos avançados de templates.

Autores e colaboradores

Você pode @mention um nome de usuário GitHub para gerar um link para seu perfil. O resultando <a> elemento vai ligar para GitHub Perfil do contribuinte. Por exemplo: Em 2007, Chris Wanstrath (@defunkt), PJ Hyett (@pjhyett), e Tom Preston-Werner (@mojombo) fundou GitHub.

Suporte ou Contato

Tendo problemas com as páginas? Confira a documentação em http://help.github.com/pages ou contate support@github.com e nós vamos ajudá-lo a resolver o problema..
